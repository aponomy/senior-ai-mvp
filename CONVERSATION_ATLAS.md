## What is Conversation Atlas?

**Conversation Atlas** turns long chats into a clear, zoomable map. Instead of scrolling through pages of text, users always see three views at once: the big picture of topics, the key points of the topic they're on, and the full messages only when needed.

Like having a table of contents, chapter summaries, and full text all visible simultaneously — but for conversations.

---

## The Three-Axis Architecture

Conversation Atlas organizes dialogue along three distinct axes:

### 1. **Time Axis**: Progressive Column Reveal
- **Stage 1** (0-1 topics): Conversation only (familiar chat interface)
- **Stage 2** (2+ topics): Key Points + Conversation columns appear
- **Stage 3** (2+ summaries): All columns visible (full Atlas interface)

Interface grows with user understanding — never overwhelming, always clear.

### 2. **Detail Axis**: Five Resolution Levels
- **Level 1**: Full detail messages (recent conversation)
- **Level 2**: Condensed/blurred boxes (50-70% reduction)
- **Level 3**: Abstract summary boxes (75-85% reduction)
- **Level 4**: Conversation summaries (entire conversation in 2-3 sentences)
- **Level 5**: Topic clusters (groups of related conversations)

Users can zoom in for detail, zoom out for overview — bidirectionally navigable.

### 3. **Intent Axis**: Three-Category Separation
- **Regular** (Blue): Content discussions (recipes, appointments, questions)
- **Functions** (Green): Action executions (bank transfers, bookings, API calls)
- **Meta** (Amber): Interface help (navigation, feature explanations)

Each category has distinct visual treatment and summarization rules.

## How It Works

### Interactive Zoom Navigation
- **Click summary** → See key points
- **Click key point** → Jump to full messages
- **Click "continue from here"** → Reactivate old context
- **Bidirectional**: Zoom in for detail, zoom out for overview

### Live Synchronization
- Three views stay synchronized
- Selecting in one updates the others
- Always see context at multiple resolutions simultaneously

### Progressive Disclosure
- Starts as simple chat (familiar, not intimidating)
- Reveals structure gradually (key points appear when helpful)
- Shows history naturally (summaries emerge as conversations age)
- Never overwhelming (complexity revealed when ready)

## Alternative Names Considered

The following names were evaluated but not selected:

1. **Progressive Disclosure UI** - Too generic, widely used
2. **Three-Column Architecture** - Too technical, focuses on structure not benefit
3. **Gradient Information** - Unclear meaning
4. **3D Conversation** - Conflicts with VR/AR concepts, implies special hardware
5. **Memory Mosaic** - Visual but less navigational
6. **Dialogue Viewfinder** - Camera metaphor, less familiar
7. **TalkLayers** - Simple but less distinctive
8. **Context Compass** - Orientation focus, less about multi-resolution
9. **Clarity Panorama** - Wide view but static feeling

**Conversation Atlas** was chosen for:
- Familiar map metaphor (elderly-friendly)
- Captures zoom in/out navigation concept
- Distinctive and brandable (ownable trademark)
- Works across technical docs, patents, marketing, and UI
- Avoids VR/spatial audio confusion
- Immediately understandable value proposition
