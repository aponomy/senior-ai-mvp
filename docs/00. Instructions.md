# EIC Application Workflow Instructions

## Purpose
These instructions guide the iterative research and writing process for completing EIC Accelerator application sections. The workflow emphasizes deep research, reasoning, and synthesis before writing final content.

---

### Phase 2: Strategic Clarification (Ongoing)

4. **Strategic Questions â†’ Ask User (as they arise)**
   - If questions concern:
     - Product vision or positioning
     - Business model decisions
     - Target market focus
     - Feature priorities
     - Go-to-market strategy
   - **Action**: Present strategic questions to user, wait for answers
   - **Save**: Document strategic decisions in `docs/decisions/[issue]_[task]_[topic]-decision.md`

### Phase 3: Deep Research (Iterative)

**âš ï¸ Only begin Phase 3 after user has answered your 3-5 clarifying questions from Phase 1**

5. **Research Planning**
   - Break down each research need into specific, answerable questions
   - Prioritize questions by importance to final output
   - Identify dependencies between questions (what must be answered first)
   - Create research roadmap

6. **First Research Iteration**
   - [ ] Ask expert about foundational questions (2-4 questions)
   - [ ] Save each Q&A in `docs/ask_expert/[issue]_[task]_[topic]-[date].md` with:
     - Date, Issue #, Task #, Purpose
     - Question with full context
     - Expert's answer
     - Key insights extracted
     - Follow-up questions identified
   
   **File Naming**: `[issue]_[task]_[topic]-[date].md`
   - Example: `10_01_Innovation-Breakthrough-Nature-2025-11-22.md`

7. **Analysis & Synthesis**
   - [ ] Review expert answers
   - [ ] Extract key facts, statistics, quotes
   - [ ] Identify contradictions or gaps in information
   - [ ] Cross-reference with existing documents
   - [ ] Determine if information is sufficient or if more research is needed
   - [ ] Document reasoning in `docs/reasoning/[issue]_[task]_[topic]-analysis.md`
     - Example: `10_01_Innovation-Breakthrough-analysis.md`

   **âš ï¸ CRITICAL CHECKPOINT - Strategic Implications Assessment**
   
   After analyzing research, **STOP and ASK USER** if you discover anything suggesting Senior AI's tech, features, or offering should be modified:
   - New competitor info affecting positioning
   - Market insights shifting target segment
   - Technology trends changing feature emphasis
   - Pricing/business model implications
   - Regulatory/compliance impacts
   - User research contradicting assumptions
   
   **Format**: Present: What I Found â†’ Current Position â†’ Implication â†’ Question with Options (A/B/C)
   
   **DO NOT proceed with writing** until strategic alignment is confirmed.

8. **Subsequent Research Iterations** (Repeat as needed)
   - [ ] Based on analysis, formulate follow-up questions
   - [ ] Ask expert about deeper or clarifying questions
   - [ ] Save new Q&As using same naming convention: `[issue]_[task]_[topic]-[date].md`
   - [ ] Synthesize new information with previous findings
   - [ ] Update reasoning document (same naming convention)
   - [ ] **Convergence check**: Can I now answer the original section requirements?
   
   **Iteration Guidelines**:
   - Typical: 2-4 iterations per complex topic
   - Each iteration should narrow focus or deepen understanding
   - Stop when: sufficient information to write confidently + defensibly

9. **Comparative Analysis** (when applicable)
   - [ ] Compare multiple sources or perspectives
   - [ ] Identify consensus vs. outlier views
   - [ ] Evaluate credibility and recency of information
   - [ ] Document comparison in `docs/reasoning/[issue]_[task]_[topic]-comparison.md`
     - Example: `10_03_Competitive-Advantages-comparison.md`

---

### Phase 4: Synthesis & Writing

10. **Create Synthesis Document**
   - [ ] Compile all research findings into `docs/synthesis/[issue]_[task]_[topic]-synthesis.md`
   - Include:
     - Key facts and statistics (with sources)
     - Main arguments and evidence
     - Competitive insights
     - Gaps that remain (if any)
     - Recommended narrative structure
   - This becomes the foundation for final writing
   - Example: `10_01_Innovation-Breakthrough-synthesis.md`

11. **Draft Application Section**
    - [ ] Write draft in `docs/drafts/[issue]_[task]_[topic]-draft.md`
    - Use synthesis document as reference
    - Follow EIC formatting requirements (from template)
    - Ensure claims are backed by research
    - Include citations where appropriate
    - Match tone: professional, compelling, evidence-based
    - Example: `10_01_Innovation-Breakthrough-draft.md`

12. **Quality Check**
    - [ ] Verify all claims have supporting evidence
    - [ ] Check for internal consistency with other sections
    - [ ] Ensure EIC evaluation criteria are addressed
    - [ ] Review against issue checklist
    - [ ] Flag any remaining uncertainties for user review

---

### Phase 5: Review & Finalization

13. **Present to User**
    - [ ] Share chat summary covering:
      - **Research conducted**: Topics explored, iterations completed
      - **Key findings**: Most important insights discovered
      - **Decisions made**: How research informed writing
      - **Draft status**: What's complete, what needs review
      - **Remaining questions**: Any strategic items for user
    - [ ] Provide links to all created documents
    - [ ] Request feedback on draft

14. **Iterate Based on Feedback**
    - [ ] Address user comments
    - [ ] Do additional research if needed
    - [ ] Refine draft
    - [ ] Update synthesis and reasoning docs

15. **Finalize**
    - [ ] Move approved content to final location
    - [ ] Update issue checklist (mark completed tasks)
    - [ ] Archive research documents for reference
    - [ ] Note any insights useful for other sections
